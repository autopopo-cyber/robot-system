# 探索狗平台 — 设计规格

> 版本: v1.0 | 日期: 2026-05-22 | 作者: 俊秀 + 主人

## 1. 目标

遥控器开进目标室内 → 信号丢失触发自主模式 → 狗遍历整个房间（体素建图 + 图像采集）→ 地图收敛后回到出发点 → 操作员取走狗和SSD → 4090笔记本出3DGS

**核心原则：**
- 运控交给宇树RL模型，平台不管运动控制
- PC2只跑一个东西：探索引擎
- 不做RTC、不做遥控站——宇树遥控器 + 5G图传已覆盖
- 数据全部存狗上SSD，事后导出

## 2. 架构

```
PC2 上只跑一个进程：exploration_engine.py

┌─────────────────────────────────────────────┐
│              exploration_engine.py           │
│                                              │
│   ┌─────────┐   ┌──────────┐   ┌──────────┐│
│   │ 信号看门狗│   │ 萤火V3    │   │ 数据记录  ││
│   │         │   │          │   │          ││
│   │ 检测RTC │──▶│ 体素建图   │──▶│ LiDAR点云 ││
│   │ 断连=触发│   │ find_gates│   │ 图像(JPG) ││
│   │ 重连=暂停│   │ A*寻路    │   │ 体素地图  ││
│   └─────────┘   │ 局部避障   │   │ 姿态log   ││
│                 └────┬─────┘   └──────────┘│
│                      │                      │
│            ┌─────────▼──────────┐            │
│            │     运动控制        │            │
│            │ /api/sport/request │◄─── DDS    │
│            │   API_ID=1008      │            │
│            └────────────────────┘            │
└──────────────────────┬──────────────────────┘
                       │ DDS
        ┌──────────────┼──────────────┐
        ▼              ▼              ▼
   LowState        LiDAR点云     宇树运控
   (IMU+电机)     (OTA后)      (PC1, 不管)
```

## 3. 模块

### 3.1 信号看门狗 (watchdog.py)

| 项目 | 描述 |
|------|------|
| 输入 | RTC连接状态（topic或进程存活检测） |
| 输出 | 触发/暂停探索状态 |
| 逻辑 | 断连超过N秒 → 启动探索；重连 → 暂停并上报状态 |

### 3.2 萤火V3引擎 (firefly_v3.py) — 从仿真移植

| 项目 | 描述 |
|------|------|
| 输入 | LiDAR点云（或USB摄像头模拟）+ LowState姿态 |
| 输出 | waypoint → 转API_ID=1008 Move指令 |
| 核心 | 体素grid(free/unknown/wall) → find_gates → A*寻路 |
| 探索策略 | 从gates中选未访问方向 → A*走过去 → LiDAR更新grid → 重复 → 无新gates=收敛 |

**探索完成条件：** 连续3次find_gates找不到新的FREE区域 → 体素地图不再增长 → A*回出发点。

### 3.3 局部避障 (obstacle_avoid.py)

| 项目 | 描述 |
|------|------|
| 输入 | LiDAR实时点云（前方扇形） |
| 输出 | 紧急停止 + 触发A*重规划 |
| 逻辑 | 前方1.5m内体素被占 → Damp → 重算路径 |

### 3.4 数据记录 (data_logger.py)

| 项目 | 描述 |
|------|------|
| 记录内容 | LiDAR点云(.ply)、USB图像(.jpg)、体素地图(.npz)、姿态轨迹(.csv) |
| 存储 | 狗上SSD，按时间戳建目录 |
| 格式 | 标准格式，4090笔记本直接用 |

## 4. 接口

### DDS 订阅（读）
- `/lf/lowstate` — IMU姿态+35电机状态
- LiDAR topic（OTA后确定具体名称）
- USB摄像头topic（`/usb_cam/image_raw`）

### DDS 发布（写）
- `/api/sport/request` — API_ID=1008 Move（位置+速度）
- `/api/sport/request` — API_ID=1001 Damp（紧急停止）

## 5. 交付分期

### 一期（立即，传感器未齐）
- 萤火V3移植到ROS2 — 用仿真体素数据跑通逻辑
- 信号看门狗框架 — 预留RTC检测接口
- 数据记录框架 — 姿势日志 + 体素导出
- **不做真机运动测试**（无龙门架安全铁律）

### 二期（传感器就绪后）
- LiDAR接入（OTA后）
- USB摄像头接入（主人买后）
- 避障模块上线
- 实机测试

## 6. 约束

| 约束 | 说明 |
|------|------|
| 运控 | 宇树RL模型，平台不接管 |
| 运动 | 无龙门架/固定装置绝对不发运动指令 |
| 语言 | Python 3.10 + rclpy |
| 部署 | PC2直接运行，不依赖外挂开发板 |
| 图传 | 宇树遥控器自带5G图传，平台不管 |
